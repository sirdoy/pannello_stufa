## Critical Integration Gaps

### Gap 1: `setroomthermpoint` — Missing `home_id` in Frontend (ENERGY-03)

**Impact:** Users cannot adjust room temperature from any UI component.

Phase 76 route validates `home_id` as required, but **6 call sites** in 4 components omit it:

| Component | Lines | Body Sent | Result |
|-----------|-------|-----------|--------|
| `RoomCard.tsx` | 103, 133, 161 | `{ room_id, mode, temp }` | 400 |
| `ThermostatCard.tsx` | 343 | `{ room_id, mode, temp }` | 400 |
| `ManualOverrideSheet.tsx` | 79 | `{ room_id, mode, temp }` | 400 |
| `ActiveOverrideBadge.tsx` | 57 | `{ room_id, mode }` | 400 |

**Additional:** `RoomCard.setModeOff()` (line 166) sends `mode: 'off'` but route restricts to `['manual', 'home']`.

**Fix:** Thread `home_id` from topology state into each POST body. Remap `mode: 'off'` to `mode: 'home'`.

### Gap 2: `setthermmode` — Missing `home_id` in Frontend (ENERGY-04)

**Impact:** Users cannot change thermostat mode (schedule/away/frost guard).

| Component | Line | Body Sent | Result |
|-----------|------|-----------|--------|
| `ThermostatCard.tsx` | 318 | `{ mode }` | 400 |
| `thermostat/page.tsx` | 241 | `{ mode }` | 400 |

**Fix:** Add `home_id: topology.home_id` to both POST bodies. `topology.home_id` is already in component state.
