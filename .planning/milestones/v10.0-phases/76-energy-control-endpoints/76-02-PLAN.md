    Keep: withAuthAndErrorHandler, success, badRequest, parseJsonOrThrow, validateRequired, validateEnum, adminDbPush, DEVICE_TYPES.

    New VALID_MODES: ['manual', 'home'] (remove 'max', 'off').

    New interface SetRoomThermPointBody adds home_id field (required from client):
    ```typescript
    interface SetRoomThermPointBody {
      home_id?: string;
      room_id?: string;
      mode?: string;
      temp?: number;
      endtime?: number;
    }
    ```

    Handler logic:
    1. Parse body, destructure {home_id, room_id, mode, temp, endtime}
    2. validateRequired(home_id, 'home_id')
    3. validateRequired(room_id, 'room_id')
    4. validateRequired(mode, 'mode')
    5. validateEnum(mode, VALID_MODES, 'mode')
    6. If mode === 'manual' && temp is undefined/null → return badRequest('temp e obbligatorio per mode=manual')
    7. Build proxyBody: { home_id, room_id, mode, ...(mode==='manual' && {temp}), ...(endtime && {endtime}) }
    8. Try: await proxySetRoomThermpoint(proxyBody) → return success({})
    9. Catch (error): log failure to Firebase then re-throw:
       ```typescript
       const logEntry = {
         action: 'Modifica temperatura stanza',
         device: DEVICE_TYPES.THERMOSTAT,
         value: temp ? `${temp}°C` : mode,
         room_id, mode, temp: temp ?? null, endtime: endtime ?? null,
         timestamp: Date.now(),
         user: user ? { email: user.email, name: user.name, picture: user.picture, sub: user.sub } : null,
         source: 'manual',
         error: error instanceof Error ? error.message : String(error),
       };
       await adminDbPush('log', logEntry);
       throw error;
       ```
  </implementation>
</feature>
