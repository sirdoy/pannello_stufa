<objective>
Create the shared HomeAssistant proxy client module with generic GET and POST helpers that all provider clients (Fritz!Box, Netatmo, Raspberry Pi) will build on.

Purpose: Eliminate duplicated fetch+timeout+error-mapping logic across provider clients. The Netatmo proxy client (lib/netatmoProxy.ts) and Fritz!Box client (lib/fritzbox/fritzboxClient.ts) both implement nearly identical X-API-Key auth, AbortController timeouts, and RFC 9457 error parsing. This plan extracts that into a single shared module.

Output: `lib/haClient.ts` with `haGet<T>()` and `haPost<T>()` functions, `types/haClient.ts` with shared types, and comprehensive unit tests.
</objective>

    ```typescript
    /**
     * Shared HomeAssistant Proxy Client
     *
     * Generic GET and POST helpers for the HomeAssistant network API proxy.
     * All providers (Fritz!Box, Netatmo, Raspberry Pi) use this as their transport.
     *
     * Configuration (env vars):
     *   HA_API_URL — Base URL of the HA proxy (e.g. https://ha.example.com)
     *   HA_API_KEY — Shared API key for X-API-Key header authentication
     *
     * Error handling:
     *   - RFC 9457 error responses parsed and mapped to ApiError instances
     *   - AbortError (timeout) -> ApiError(TIMEOUT)
     *   - 401 -> ApiError(UNAUTHORIZED)
     *   - 429 -> ApiError(RATE_LIMITED)
     *   - 503 -> ApiError(SERVICE_UNAVAILABLE)
     *   - Other non-ok -> ApiError(EXTERNAL_API_ERROR)
     */

    import { ApiError, ERROR_CODES, HTTP_STATUS } from '@/lib/core/apiErrors';
    import type { RFC9457ProblemDetail, HaRequestOptions } from '@/types/haClient';

    const DEFAULT_TIMEOUT_MS = 15_000;
    ```

    Implement `haGet<T>(endpoint: string, options?: HaRequestOptions): Promise<T>`:
    - Read HA_API_URL and HA_API_KEY from process.env
    - If either missing, throw `new ApiError(ERROR_CODES.EXTERNAL_API_ERROR, 'HA proxy not configured: missing {varName}', HTTP_STATUS.INTERNAL_SERVER_ERROR)`
    - Create AbortController, set timeout via setTimeout
    - Fetch `${baseUrl}${endpoint}` with headers `{ 'X-API-Key': apiKey }` and signal
    - On success: clearTimeout, return `(await response.json()) as T`
    - On !response.ok: parse RFC 9457 body, map status 401->UNAUTHORIZED, 429->RATE_LIMITED, 503->SERVICE_UNAVAILABLE, else->EXTERNAL_API_ERROR
    - Catch: clearTimeout, re-throw ApiError as-is, map AbortError->ApiError.timeout('HA proxy timeout'), unknown->EXTERNAL_API_ERROR

    Implement `haPost<T>(endpoint: string, body: Record<string, unknown>, options?: HaRequestOptions): Promise<T>`:
    - Same env var validation and AbortController setup as haGet
    - Fetch with method 'POST', headers `{ 'X-API-Key': apiKey, 'Content-Type': 'application/json' }`, body: JSON.stringify(body)
    - Same error handling as haGet

    Extract common error-handling into a private `mapResponseError` helper to avoid duplication between GET and POST (unlike netatmoProxy.ts which duplicates — this is the improvement).

    Export only: `haGet`, `haPost`.

    Run tests — all MUST pass (GREEN phase).
