  <action>
Root cause: `screen.getByText('ACCENDI')` resolves to the inner `<span>{children}</span>` rendered by Button.tsx, not the `<button>` element. `toBeDisabled()` on a plain `<span>` always fails — it is not a form control and does not inherit the button's `disabled` attribute.

Fix: In the 3 disable-state tests only, replace `screen.getByText('ACCENDI')` with `screen.getByRole('button', { name: /ACCENDI/i })`. This queries by ARIA role, which traverses to the actual `<button>` element.

The 6 passing tests use `screen.getByText()` for click/presence assertions — leave those unchanged. Only the 3 `toBeDisabled()` tests need updating.

Exact changes (3 tests):

Test: "disables ACCENDI when needsMaintenance is true" (line 48–52)
- Change: `const button = screen.getByText('ACCENDI');`
- To:     `const button = screen.getByRole('button', { name: /ACCENDI/i });`

Test: "disables buttons when loading is true" (line 54–58)
- Change: `const button = screen.getByText('ACCENDI');`
- To:     `const button = screen.getByRole('button', { name: /ACCENDI/i });`

Test: "disables buttons when command is executing" (line 60–64)
