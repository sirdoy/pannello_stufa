### Integration Findings (Info-level only)

**INT-HISTORY:** History route has no frontend consumer (info)
- `GET /api/hue/history` exists and is tested but no UI page fetches it.
- **Impact:** Infrastructure ready for future analytics. Route is functional and tested.

**INT-DEBUG-BRIDGE-STATUS:** Debug HueTab checks wrong field for bridge status (info)
- Both HueTab files check `data.bridgeConnected` but proxy response field is `connected`.
- Bridge Status badge permanently hidden. Fix: rename to `data.connected`.
- **Impact:** Debug panel only. Production `useLightsData` reads correct field.

**INT-DEBUG-BRIGHTNESS-KEY:** Debug HueTab sends wrong body key for brightness (info)
- Both HueTab files send `{ brightness: values.brightness }` but proxy expects `bri`.
- Brightness changes via debug panel have no effect; on/off works correctly.
- **Impact:** Debug panel only. Production hooks use correct `bri` key.
