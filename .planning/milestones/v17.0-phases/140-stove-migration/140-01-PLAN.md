<objective>
Migrate useStoveData from pure HTTP polling to WebSocket-primary with HTTP polling fallback.

Purpose: The stove card currently polls every 60 seconds. With WebSocket as primary channel, state updates arrive within ~1 second. HTTP polling automatically activates as fallback when the WS connection is unavailable, preserving alwaysActive behavior for safety-critical stove monitoring.

Output: Modified useStoveData.ts with WS subscription + polling fallback, updated test suite with WS mock coverage.
</objective>
