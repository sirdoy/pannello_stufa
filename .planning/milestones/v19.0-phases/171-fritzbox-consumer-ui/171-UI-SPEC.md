# Phase 171 тАФ UI Design Contract

> Visual and interaction contract for Fritz!Box Consumer UI (telephony, raw history, service discovery). Generated by gsd-ui-researcher, verified by gsd-ui-checker. Phase 171 is pure UI wiring on top of existing API routes (shipped in phase 162) and an already-mature design system (Ember Noir v3.0). No new primitives, no new CVA variants, no new dependencies.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (in-house Ember Noir v3.0 тАФ `docs/design-system.md`) |
| Preset | not applicable (shadcn not used in this project) |
| Component library | in-house primitives in `app/components/ui/*` (Radix UI underneath for Checkbox/Switch/Tabs; TanStack Table underneath for DataTable) |
| Icon library | lucide-react ^0.562.0 |
| Font | `Outfit` (display/headings) + `Space Grotesk` (body), self-hosted via `next/font` (Phase 70) |
| Theme | Ember Noir тАФ dark-first only (light overrides shipped in primitives; page code never writes raw colors) |

**Scope note:** Phase 149 removed runtime theme switching; Phase 151 enforced mobile-first. Phase 171 MUST NOT introduce theme tokens, raw color classes, or new CVA variants. Use component `variant` props only.

## Spacing Scale

Declared values (8-point, multiples of 4, matching Ember Noir tokens):

| Token | Value | Tailwind | Usage in Phase 171 |
|-------|-------|----------|--------------------|
| xs | 4px | `gap-1`, `p-1` | Inline icon-to-label spacing (call type badge icon) |
| sm | 8px | `gap-2`, `p-2` | Compact Badge padding, toolbar button gaps |
| md | 16px | `gap-4`, `p-4` | Card inner padding on mobile, DataTable cell padding |
| lg | 24px | `gap-6`, `p-6` | Card inner padding on desktop (`sm:p-6`), section vertical spacing |
| xl | 32px | `gap-8`, `p-8` | Between the three `/telefonia` stacked sections |
| 2xl | 48px | `gap-12` | Between tab content and next page block (reuse of existing `/network` spacing) |

**Exceptions:** 44px minimum touch-target for Prev/Next pagination buttons and the refresh button (WCAG 2.5.5). Enforced by `<Button size="sm">` which already ships at 40px height + padding тЖТ 44px effective. No bespoke exceptions.

**Card padding pattern:** `p-4 sm:p-6` (mobile-first) тАФ identical to `DeviceListTable.tsx:165`.

**Page shell pattern:** `<PageLayout maxWidth="7xl">` тЖТ `space-y-6` for vertical rhythm (identical to `/network/page.tsx:200` and `/sonos/page.tsx:60`).

## Typography

All sizes expressed as the canonical `size` prop surface of `Heading` / `Text` primitives (never raw `text-xl` classes). Only 4 sizes + 2 weights appear in this phase.

| Role | Primitive | Size prop | Approx px | Weight | Line Height | Use |
|------|-----------|-----------|-----------|--------|-------------|-----|
| Display (page title) | `Heading level={1}` | `size="2xl"` | 28тАУ30px | semibold 600 (Outfit) | 1.2 | `/telefonia` H1 "Telefonia" |
| Section heading | `Heading level={2}` | `size="lg"` | 20px | semibold 600 (Outfit) | 1.2 | "Segreteria", "Cornette DECT", "Cronologia chiamate", "Storico grezzo", "Service Discovery" |
| Body / table cell | `Text variant="body"` (default) | `size="sm"` or default | 14тАУ16px | regular 400 (Space Grotesk) | 1.5 | DataTable cell text, Banner description, EmptyState description |
| Label / caption | `Text variant="secondary"` or `variant="tertiary"` | `size="xs"` | 12px | regular 400 (Space Grotesk) | 1.4 | Column headers, pagination counters, timestamp lines, "Fetched at тАж" |

**TAM new-message count exception:** The large number in the TAM card uses `<Heading level={3} size="3xl">` (~32px) with `variant="ember"` when `new_messages > 0` and default variant otherwise. This mirrors the maintenance-alert treatment on `/stove` (referenced in 171-CONTEXT.md ┬зspecifics).

**Monospace:** phone numbers and TR-064 service URLs use the existing `font-mono` utility via `<span className="font-mono text-sm text-slate-400">тАж` тАФ identical to `DeviceListTable.tsx:59` (MAC address rendering). Justified for alignment of digit-heavy strings.

## Color

60/30/10 split inherited from Ember Noir. Phase code never writes raw hex or color utility classes; all color flows through primitive `variant` props.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `slate-950` / `slate-900` (`#0c0a09` / `#1c1917`) | Page background (from `PageLayout`), DataTable page chrome |
| Secondary (30%) | `slate-800/50` + `white/[0.06]` borders | `Card` surfaces, DataTable header (`bg-slate-800/50`), row borders |
| Accent (10%) | `ember-400` / `ember-500` (`#f18d33` / `#ed6f10`) | Active tab underline, Prev/Next selected page, TAM new-messages count (when >0), sortable column sort arrows |
| Semantic тАФ danger | `danger-500` (`#ef4444`) via `<Badge variant="danger">` | "Persa" call badge, battery <20% badge, error Banner |
| Semantic тАФ warning | `warning-500` (`#eab308`) via `<Badge variant="warning">` or `<Banner variant="warning">` | Stale data Banner, battery 20тАУ49% badge, DECT "non registrato" badge |
| Semantic тАФ success | `sage-500` (`#607360`) via `<Badge variant="sage">` | "In entrata" call badge, DECT "registrato" badge |
| Semantic тАФ info | `ocean-500` (`#437dae`) via `<Badge variant="ocean">` | "In uscita" call badge, "connected" device-event badge |
| Neutral | `slate-500/10` via `<Badge variant="neutral">` | "Sconosciuto" call type fallback, "disconnected" event badge (dimmer than danger тАФ device leaving is not an error) |

**Accent reserved for (explicit list тАФ never "all interactive elements"):**
1. Active page tab underline inside `/network` tab bar ("Storico grezzo" when selected) and `/debug` `Tabs.Trigger` (existing behavior).
2. DataTable column sort arrow when a column is sorted (existing `DataTable` CVA behavior).
3. Pagination: currently-selected Prev/Next page button (`<Button variant="ember">`).
4. TAM `new_messages` large number when `> 0`.
5. Primary refresh CTA (`<Button variant="ember">`) тАФ ONLY the service-discovery manual refresh button. All other refresh uses `variant="outline"`.
6. Badge `variant="ember"` + `pulse` on DECT handset when the TAM has just rung (NOT used in this phase тАФ listed here as prohibited so executor doesn't invent it).

**Destructive color usage:** Phase 171 has NO destructive actions (read-only UI per phase boundary, D-06, ┬зdeferred in CONTEXT.md). The `danger` color appears only as semantic status (missed call, low battery) тАФ never on buttons, never with `<Button variant="danger">`.

**Call-type badge logic (exact mapping, icon + label):**

| `call_type` value | Badge variant | Icon (lucide, 14px) | Label (it) |
|-------------------|---------------|---------------------|------------|
| `"incoming"` | `sage` | `<PhoneIncoming size={14} />` | In entrata |

| `"missed"` | `danger` | `<PhoneMissed size={14} />` | Persa |
| `"voicemail"` | `warning` | `<Voicemail size={14} />` | Segreteria |
| any other / `undefined` | `neutral` | `<Phone size={14} />` | Sconosciuto |

Rendered as: `<Badge variant={v} size="sm" icon={<Icon />}> {label} </Badge>` тАФ icon slot is already baked into `Badge` per design-system.md ┬зColors.

**Duration format:**
- `duration_seconds < 3600` тЖТ `mm:ss` (e.g. `02:45`)
- `duration_seconds >= 3600` тЖТ `hh:mm:ss` (e.g. `1:12:08`)
- `duration_seconds === 0` тЖТ `<span className="text-slate-500">тАФ</span>` (unanswered)

**Row sort order:** Most recent first (`timestamp` desc). Server-paginated, so client does NOT resort across pages; within a page, the server ordering is trusted.

**Pagination state (client-side):**
- `limit = 50` (fixed; D-05)
- `offset = 0` initially
- Prev тЖТ `setOffset(Math.max(0, offset - limit))`
- Next тЖТ `setOffset(offset + limit)` (hook guards with `total_count`)
- Any change resets scroll position to top of DataTable: `tableRef.current?.scrollIntoView({ behavior: 'smooth', block: 'nearest' })`

**Empty state (no calls ever recorded, `total_count === 0`):**
```jsx
<EmptyState
  icon={<History size={48} className="text-slate-500" />}
  title="Nessuna chiamata registrata"
  description="Le chiamate appariranno qui non appena il Fritz!Box le registra."
  size="md"
/>
```

**Empty page (filtered to a page beyond data):** impossible with Prev/Next guards; no treatment needed.

**Loading state (first load):** `<Skeleton className="h-[520px] rounded-2xl" />`.
**Loading state (page transition):** keep previous rows visible, add `opacity-60 pointer-events-none` to the DataTable container and show a small `<Spinner size="sm" />` next to the count Badge. Avoids layout shift.
**Error state:** `<Banner variant="error" title="Impossibile caricare la cronologia" description={error.message} compact={true} />` replaces the DataTable and pagination footer.

**Mobile (<640px):** DataTable horizontally scrolls inside its `overflow-x-auto` container (already built into `DataTable.tsx:499`). The right-edge fade gradient indicator also ships from the primitive. No column-hiding logic.
