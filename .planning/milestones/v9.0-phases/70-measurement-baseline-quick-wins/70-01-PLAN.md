<task type="auto">
  <name>Task 2: Create reusable baseline capture and comparison script</name>
  <files>scripts/baseline.mjs</files>
  <action>
Create `scripts/baseline.mjs` — a Node.js script that:

1. **Captures bundle sizes** from `.next/build-manifest.json`:
   - Parse the manifest JSON
   - For each route (page key), sum the sizes of all JS chunks (read actual file sizes from `.next/static/` using `fs.statSync`)
   - Calculate "shared" chunks (chunks that appear in multiple routes)
   - Output a `routes` object with per-route First Load JS in bytes and a `shared` value

2. **Optionally captures Lighthouse scores** (via `--lighthouse URL` flag):
   - Run `npx lighthouse URL --output=json --quiet --chrome-flags='--headless --no-sandbox'`
   - Parse the JSON output and extract: performance score, LCP, FCP, CLS, INP, TTFB
   - If `--lighthouse` is not provided, set `lighthouse: null` in the output (bundle-only baseline)

3. **Writes a JSON snapshot** to `.baseline/phase-{PHASE}.json`:
   - Create `.baseline/` directory if it doesn't exist (`mkdirSync({ recursive: true })`)
   - JSON structure:
     ```json
     {
       "phase": "70",
       "timestamp": "2026-02-18T...",
       "bundle": {
         "routes": { "/": 120000, "/stove": 185000, ... },
         "shared": 85000,
         "totalFirstLoad": { "/": 205000, "/stove": 270000, ... }
       },
       "lighthouse": { "performance": 72, "lcp": 2800, ... } | null
     }
     ```

4. **Comparison mode** (`--compare` flag):
   - Read `.baseline/phase-70.json` as the reference baseline
   - Read current phase baseline
   - Print a formatted delta table to stdout:
     ```
     Route          | Phase 70  | Phase 71  | Delta    | %
     /              | 120.0 kB  | 105.0 kB  | -15.0 kB | -12.5%
     /stove         | 185.0 kB  | 170.0 kB  | -15.0 kB | -8.1%
     ```
   - If Lighthouse data exists in both, also print Lighthouse comparison

5. **Usage** (document in script header comment):
   ```
   PHASE=70 node scripts/baseline.mjs                    # capture bundle baseline
   PHASE=70 node scripts/baseline.mjs --lighthouse URL   # capture with Lighthouse
   PHASE=71 node scripts/baseline.mjs --compare          # compare vs phase 70
   ```

Use ES module syntax (import/export). Use only Node.js built-in modules (fs, path, child_process). No external dependencies.

Add a shebang line: `#!/usr/bin/env node`

Handle errors gracefully:
- If `.next/build-manifest.json` doesn't exist, print error and exit 1
- If `--compare` but `.baseline/phase-70.json` doesn't exist, print helpful message
- If lighthouse CLI fails, log warning and continue with `lighthouse: null`
  </action>
  <verify>
Run `node scripts/baseline.mjs --help 2>&1 || true` (script should handle unknown args gracefully). Verify the file exists and is valid JavaScript: `node --check scripts/baseline.mjs`.
  </verify>
