<objective>
Wrap all four chart components in `React.memo` and add `isAnimationActive={false}` to every Recharts series element (Bar, Line) that is missing it. This prevents chart SVG re-renders when parent components re-render on polling ticks, and eliminates visible animation restarts.

Purpose: Charts currently re-render their full SVG DOM every time a parent re-renders (e.g., on a 30-second polling tick). `React.memo` blocks prop-unchanged re-renders, and `isAnimationActive={false}` prevents animation restart flicker when data does update.

Output: Four chart components wrapped in `React.memo` with all series elements disabling animation.
</objective>

`NetworkBandwidth` is a pure presentational component that receives `bandwidth`, `downloadHistory`, and `uploadHistory` props from `NetworkCard`. The `NetworkCard` re-renders on every 30-second polling tick, which causes `NetworkBandwidth` to re-render even when its sparkline data has not changed. The sparklines already have `isAnimationActive={false}` (confirmed) so only the `React.memo` wrapper is needed.

Wrap the component in `React.memo`:

Change from:
```tsx
export default function NetworkBandwidth({ bandwidth, downloadHistory, uploadHistory }: NetworkBandwidthProps) {
```
to:
```tsx
import { memo, useId } from 'react'; // Add memo to the existing useId import from react

const NetworkBandwidth = memo(function NetworkBandwidth({ bandwidth, downloadHistory, uploadHistory }: NetworkBandwidthProps) {
  // ... existing component body ...
});
export default NetworkBandwidth;
```

The component already imports `useId` from `react`. Add `memo` to that same import statement.

NOTE: `downloadHistory` and `uploadHistory` arrays get new references on every poll tick (via `setState(prev => [...prev, newPoint])`). `React.memo` will still re-render when these arrays change — which is correct behavior (new data should render). The benefit is that when the PARENT re-renders for OTHER reasons (e.g., WAN status change, device list change), the bandwidth sparklines will NOT re-render if the arrays haven't changed. This is the expected optimization — we are NOT trying to skip renders when new bandwidth data arrives.
