<objective>
Add an `initialDelay` parameter to `useAdaptivePolling` so dashboard cards stagger their first API calls on mount, then add debounced setpoint writes to the thermostat card so rapid button clicks fire at most one API call per 500ms.

Purpose: On dashboard load, all six device cards currently fire their initial API calls within the same event loop tick, creating a thundering herd. Adding `initialDelay` spreads initial fetches across 500ms. The thermostat +/- 0.5 degree buttons currently fire an API call on every click — debouncing reduces Netatmo API pressure during rapid adjustments.

Output: Modified `useAdaptivePolling` hook with `initialDelay`, updated consumer hooks, and thermostat debounced writes.
</objective>
