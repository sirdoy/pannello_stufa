Update the `renderCard` helper to wrap each card in a Suspense boundary:

```tsx
const renderCard = (card: typeof visibleCards[number], flatIndex: number) => {
  const CardComponent = CARD_COMPONENTS[card.id];
  const CardSkeleton = CARD_SKELETONS[card.id];
  if (!CardComponent) return null;
  return (
    <div
      key={card.id}
      className="animate-spring-in transition-all duration-300 ease-out"
      style={{ animationDelay: `${flatIndex * 100}ms` }}
    >
      <DeviceCardErrorBoundary
        deviceName={DEVICE_META[card.id]?.name ?? card.id}
        deviceIcon={DEVICE_META[card.id]?.icon ?? '⚠\uFE0F'}
      >
        <Suspense fallback={CardSkeleton ? <CardSkeleton /> : null}>
          <CardComponent />
        </Suspense>
      </DeviceCardErrorBoundary>
    </div>
  );
};
```

**Key detail:** `DeviceCardErrorBoundary` wraps OUTSIDE the Suspense boundary (error boundaries should catch Suspense errors). `Suspense` wraps INSIDE the error boundary, around the card component.

**Order matters for SUSP-03:** Stove card is already first in the `visibleCards` array (sorted by `order` field in deviceConfig, stove is order 0). Combined with Firebase `onValue()` having no HTTP delay (vs 50-500ms stagger on other cards), stove naturally appears first. No additional prioritization logic needed.
