**File 1: `__mocks__/react-use-websocket.ts`**

Create a Jest mock for react-use-websocket that allows tests to control `sendJsonMessage`, `lastMessage`, and `readyState`.

```typescript
import { jest } from '@jest/globals';

export enum ReadyState {
  CONNECTING = 0,
  OPEN = 1,
  CLOSING = 2,
  CLOSED = 3,
}

const mockSendJsonMessage = jest.fn();

let mockReadyState = ReadyState.OPEN;
let mockLastMessage: { data: string } | null = null;
let mockOnOpen: (() => void) | null = null;

export const useWebSocket = jest.fn((
  _url: string | null,
  options?: { onOpen?: () => void; shouldReconnect?: () => boolean; reconnectAttempts?: number; reconnectInterval?: (attempt: number) => number },
  _connect?: boolean
) => {
  if (options?.onOpen) {
    mockOnOpen = options.onOpen;
  }
  return {
    sendJsonMessage: mockSendJsonMessage,
    lastMessage: mockLastMessage,
    readyState: mockReadyState,
  };
});

// Test helpers — exported for test control
export const __mockHelpers = {
  getSendJsonMessage: () => mockSendJsonMessage,
  setReadyState: (state: ReadyState) => { mockReadyState = state; },
  setLastMessage: (msg: { data: string } | null) => { mockLastMessage = msg; },
  triggerOnOpen: () => { mockOnOpen?.(); },
  reset: () => {
    mockSendJsonMessage.mockClear();
    mockReadyState = ReadyState.OPEN;
    mockLastMessage = null;
    mockOnOpen = null;
    (useWebSocket as jest.Mock).mockClear();
  },
};

export default useWebSocket;
```

**File 2: `__tests__/hooks/useWebSocketManager.test.ts`**

Write unit tests for `useWebSocketManager` covering all 5 behavioral requirements:

**Test: WS-01 — Single connection** — Calling `useWebSocketManager` calls `useWebSocket` exactly once with the provided URL. Verify `useWebSocket` was called with the URL string as first arg and options containing `shouldReconnect`, `reconnectAttempts`, `reconnectInterval`.

**Test: WS-02 — Subscribe sends message** — Call `manager.subscribe('fritzbox', callback)`. Verify `sendJsonMessage` was called with `{ action: 'subscribe', topic: 'fritzbox' }`.

**Test: WS-02 — Unsubscribe sends message when last callback** — Subscribe two callbacks for 'fritzbox'. Unsubscribe one — verify NO unsubscribe message sent. Unsubscribe the second — verify `sendJsonMessage` called with `{ action: 'unsubscribe', topic: 'fritzbox' }`.

**Test: WS-03 — Dispatch by topic** — Subscribe a callback to 'fritzbox' and another to 'hue'. Simulate an incoming message `{ type: 'event', topic: 'fritzbox', data: { devices: [] }, ts: 123 }` via the lastMessage effect. Verify the fritzbox callback was called with `{ devices: [] }` and the hue callback was NOT called.

**Test: WS-04 — Reconnection config** — Verify `useWebSocket` was called with options containing `reconnectAttempts: 10` and that `reconnectInterval` is a function. Call the function with attempt=0 and verify result is 1000. Call with attempt=5 and verify result is 30000 (capped).

**Test: WS-05 — Re-subscribe on reconnect** — Subscribe to 'fritzbox' and 'sonos'. Clear sendJsonMessage mock. Call `triggerOnOpen()` to simulate reconnect. Verify `sendJsonMessage` was called twice: once with `{ action: 'subscribe', topic: 'fritzbox' }` and once with `{ action: 'subscribe', topic: 'sonos' }`.

**Test: Null URL disables connection** — Call `useWebSocketManager(null)`. Verify `useWebSocket` was called with `null` as first arg and `false` as third arg (connect disabled).

Use `renderHook` from `@testing-library/react` and `act` from `react` for hook testing. Import `useWebSocketManager` from `@/lib/hooks/useWebSocketManager`. Use the `__mockHelpers` from the mock to control state.

Note: For the dispatch test (WS-03), the mock needs to return the lastMessage. The approach is:
1. Render the hook
2. Update the mock's lastMessage via `__mockHelpers.setLastMessage`
3. Re-render the hook to trigger the useEffect
4. Assert the callback was called
