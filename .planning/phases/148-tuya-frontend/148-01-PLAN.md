    useTuyaData:
    - Returns { plugs, loading, error, stale, lastUpdatedAt }
    - When WS connected: subscribes to 'tuya' topic, updates plugs from WS payload
    - When WS disconnected: polls GET /api/tuya/plugs at 60s interval (visible) / 300s (hidden)
    - Sets stale=true when data_freshness !== 'LIVE' or fetch fails
    - Sets loading=false after first data arrives (WS or fetch)
    - Guards against null plugs in WS payload (TuyaData.plugs can be null)

    useTuyaCommands:
    - togglePlug(deviceId, currentState): POST /api/tuya/plugs/{deviceId}/state with { on: !currentState }
    - setTimer(deviceId, seconds): POST /api/tuya/plugs/{deviceId}/timer with { seconds }
    - cancelTimer(deviceId): POST /api/tuya/plugs/{deviceId}/timer with { seconds: 0 }
    - Returns optimistic-updated plug on success (from TuyaPlugMutation)
    - When data_confirmed=false, returns null to signal caller should re-fetch after 2s
  </behavior>
