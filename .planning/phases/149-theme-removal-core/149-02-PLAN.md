  <files>app/globals.css</files>
  <read_first>app/globals.css</read_first>
  <action>
    **Step 1 — Remove the theme-switching transition rule:**
    Delete the block at approximately lines 343-346:
    ```css
    /* Smooth transitions for theme changes */
    * {
      @apply transition-colors duration-200;
    }
    ```
    Also delete the associated `.no-transition` block (lines 348-350) since it only existed to disable the theme transition:
    ```css
    .no-transition {
      transition: none !important;
    }
    ```

    **Step 2 — Remove ALL 26 `html:not(.dark)` selector blocks:**
    These are scattered throughout globals.css. Each `html:not(.dark)` block defines light-mode overrides. Delete each block entirely (the selector, its contents, and closing brace). The blocks include:

    1. `html:not(.dark) body { background: linear-gradient(...) }` (lines ~332-341)
    2. `html:not(.dark) ::-webkit-scrollbar-thumb { ... }` (line ~393)
    3. `html:not(.dark) ::-webkit-scrollbar-thumb:hover { ... }` (line ~398)
    4. `html:not(.dark) { ... }` — large semantic overrides block (starts ~line 404, contains all CSS custom property overrides for light mode)
    5. Individual class overrides: `.heading-1` through `.caption` (lines ~456-537)
    6. Card variants: `.card-ember`, `.card-ember:hover`, `.card-ember-elevated`, `.card-subtle` (lines ~553-600)
    7. Button variants: `.btn-subtle`, `.btn-subtle:hover`, `.btn-ghost`, `.btn-ghost:hover` (lines ~628-666)
    8. Status variants: `.status-active`, `.status-inactive`, `.status-success`, `.status-warning`, `.status-error` (lines ~694-758)
    9. Glass dark variant: `.glass-dark` (line ~1200)

    Use grep to find every `html:not(.dark)` occurrence and delete its complete rule block (from selector through closing `}`).

    **Step 3 — Remove any comments that reference light mode or theme switching:**
    Delete comments like `/* Light mode - Warm cream background */`, `/* Smooth transitions for theme changes */`, or any comment block that references light/dark switching behavior. Leave comments that document dark-mode styling.

    **IMPORTANT:** Do NOT delete any CSS rules that do NOT contain `html:not(.dark)`. The dark-mode rules (those without the `html:not(.dark)` prefix) must all remain intact.

    **IMPORTANT:** Do NOT delete the `body` base styles — only delete the `html:not(.dark) body` override block.
  </action>
