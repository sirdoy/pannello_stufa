| Token | Value | Usage |

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label / caption | 14px | 400 (regular) | 1.4 |
| Heading (page, modal title) | 20px | 600 (semibold) | 1.2 |
| Display (page h1) | 28px | 600 (semibold) | 1.2 |

Implementation: Use `<Heading>` and `<Text>` design system components exclusively. Never apply raw Tailwind typography classes.

```jsx
// Page title
<Heading level={1} variant="ember">Automazioni</Heading>

// Modal title
<Heading level={2}>Nuova Automazione</Heading>

// Table cell body
<Text variant="body">Nome regola</Text>

// Secondary info / caption
<Text variant="secondary" size="sm">2026-04-08 14:32</Text>
```

Source: `docs/design-system.md` Typography section. h1=3xl (30-36px mapped to 28px display), h3=xl (20-24px mapped to 20px heading). Two weights: 400 regular + 600 semibold.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | slate-950 `#0c0a09` / slate-900 `#1c1917` | Page background, DataTable row backgrounds |
| Secondary (30%) | slate-800/80 with backdrop-blur | Cards, modal overlays, nav dropdown, table header |
| Accent (10%) | ember-500 `#ed6f10` / ember-400 `#f18d33` | See reserved list below |
| Destructive | danger-500 `#ef4444` | Delete rule button, ConfirmationDialog destructive state |

**Accent reserved for:**
- Page title variant (`<Heading variant="ember">Automazioni</Heading>`)
- Primary CTA button — "Nuova Regola" (`<Button variant="ember">`)
- Active/enabled rule badge (`<Badge variant="ember" pulse>Attiva</Badge>`)
- Execution status badge when last run succeeded (`<Badge variant="ember">`)
- Focus ring on form inputs (already built into design system)

**Not** used for: table row hover states, secondary badges, read-only text fields.

Semantic colors:
- sage-500 `#607360` — `<Badge variant="sage">Completata</Badge>` for successful execution status
- danger-500 `#ef4444` — `<Badge variant="danger">Fallita</Badge>` for failed execution status + delete confirmations
- warning-500 `#eab308` — `<Badge variant="warning">In esecuzione</Badge>` for in-progress execution status
- neutral — `<Badge variant="neutral">Disattiva</Badge>` for disabled/inactive rules

Source: `docs/design-system.md` Colors section + existing pattern in `app/registry/devices/page.tsx` (getProviderBadgeVariant).

## Page Layout Contract

### `/automations` — Rules List Page

```
PageLayout
└── SettingsLayout
    ├── PageLayout.Header
    │   ├── title: "Automazioni"  (Heading level=1 variant="ember")
    │   ├── description: "Gestisci le regole di automazione del sistema"  (Text secondary)
    │   └── actions: <Button variant="ember" icon="+" iconPosition="left">Nuova Regola</Button>
    ├── Banner variant="danger"  (shown only on load error)
    ├── Skeleton (shown during loading, 5 rows)
    └── DataTable (shown when loaded)
        └── columns: [Nome, Stato, Ultima esecuzione, Azioni]
```

DataTable columns:

| Column | Content | Width |
|--------|---------|-------|
| Nome | Rule name as `<Text>` + optional description as `<Text variant="secondary" size="sm">` | flex-1 |
| Stato | `<Badge>` — ember/neutral for attiva/disattiva | 100px |
| Ultima esecuzione | ISO timestamp formatted as relative time (`<Text variant="secondary" size="sm">`) or "—" if never run | 180px |
| Azioni | `<Button.Icon>` edit + `<Button.Icon>` delete, 44px touch targets | 100px |

Pagination: uses DataTable built-in pagination, page size 20 (matches PAGE_SIZE in devices page).

### `/automations/[rule_id]` — Rule Detail + Execution History Page

```
PageLayout
└── SettingsLayout
    ├── PageLayout.Header
    │   ├── title: rule.name  (Heading level=1)
    │   ├── description: rule.description or "Nessuna descrizione"  (Text secondary)
    │   └── actions: Back button + Edit button
    ├── Card variant="elevated"  (rule metadata summary)
    │   └── Card.Content: stato, created_at, rule_id
    └── Section title="Storico Esecuzioni"
        └── DataTable (paginated execution history)
            └── columns: [Data, Stato, Durata, Dettaglio]
```

Execution history DataTable columns:

| Column | Content | Width |
|--------|---------|-------|
| Data | ISO timestamp formatted local datetime | 180px |
| Stato | `<Badge>` — sage/danger/warning for completata/fallita/in esecuzione | 120px |
| Durata | Duration in ms or "—" | 100px |
| Dettaglio | Short error message or "—" as `<Text variant="secondary" size="sm">` | flex-1 |

## Interaction Contract

### Destructive Actions

1. **Delete rule** — `<ConfirmationDialog>` blocking modal required before DELETE call.
   - Trigger: delete icon button in table row Actions column
   - Dialog renders rule name in body copy (interpolated)
   - Confirm button: `<Button variant="danger">Elimina</Button>`
   - Cancel button: `<Button variant="subtle">Annulla</Button>`
   - On confirm: DELETE /api/v1/automations/{rule_id} → success toast → refetch list
   - Pattern: matches `app/rooms/page.tsx` ConfirmationDialog usage

### CRUD Flow

- **Create:** "Nuova Regola" button → FormModal opens → form filled → Zod validation → POST /api/v1/automations → success toast + modal closes + list refetches
- **Edit:** edit icon in row → FormModal opens pre-populated → Zod validation → PATCH /api/v1/automations/{rule_id} → success toast + modal closes + list refetches
- **View detail/history:** click on rule name row → navigate to `/automations/[rule_id]`
- **Pagination:** DataTable internal, page size 20, `limit`+`offset` query params to API

### Form Fields (FormModal)

The frontend proxies whatever the HA schema exposes. Minimum viable fields:

| Field | Component | Validation |
|-------|-----------|------------|
| Nome (name) | `<Input label="Nome">` | required, min 1, max 128 |
| Descrizione (description) | `<Input label="Descrizione">` | optional, max 500 |
| Abilitata (enabled) | `<Switch label="Abilitata">` | boolean, default true |

Additional structured fields (triggers, conditions, actions) deferred to HA schema discovery at implementation time. If the HA API returns a complex JSON structure, render as a `<Textarea>` with JSON validation via Zod `.string().refine(isValidJson)` — not a visual rule editor (CONTEXT.md D-05).

### Navigation

Add "Automazioni" as a top-level nav entry using `<DropdownItem>` (matches Stanze, Registro entries). Icon: use a gear/automation icon from lucide-react (e.g. `Zap` or `Workflow`). Route: `/automations`.

Source: CONTEXT.md D-07.
