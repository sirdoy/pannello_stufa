Add a zod schema inline in the route file and use `.parse()` to narrow the body. The `parseJson` helper returns `Record<string, unknown>` but `createAutomation` expects `AutomationCreate`. Add zod parsing:

```typescript
import { z } from 'zod';
// ... existing imports ...

const automationCreateSchema = z.object({
  name: z.string().min(1),
  description: z.string().nullable().optional(),
  enabled: z.boolean().optional(),
});

export const POST = withAuthAndErrorHandler(async (request) => {
  const body = await parseJson(request);
  const parsed = automationCreateSchema.parse(body);
  const data = await automationsProxy.createAutomation(parsed);
  return created(data as unknown as Record<string, unknown>);
}, 'Automations/Create');
```

Note: `z.parse()` throws `ZodError` on invalid input. Check if `withAuthAndErrorHandler` in `lib/core/middleware.ts` catches generic errors and returns 500. If it does (which is the standard pattern), `ZodError` will be caught and returned as a 500. To get a proper 400 instead, wrap in try/catch:

```typescript
export const POST = withAuthAndErrorHandler(async (request) => {
  const body = await parseJson(request);
  let parsed: z.infer<typeof automationCreateSchema>;
  try {
    parsed = automationCreateSchema.parse(body);
  } catch (err) {
    if (err instanceof z.ZodError) {
      return badRequest(err.issues.map(i => i.message).join(', '));
    }
    throw err;
  }
  const data = await automationsProxy.createAutomation(parsed);
  return created(data as unknown as Record<string, unknown>);
}, 'Automations/Create');
```
