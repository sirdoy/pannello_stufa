  <action>
Create `lib/netatmoProxy.ts` as a simple function module (NOT a class, per user decision):

1. `netatmoProxyGet<T>(endpoint: string, options?: { timeout?: number }): Promise<T>` — the core function:
   - Read `NETATMO_PROXY_URL` and `NETATMO_PROXY_API_KEY` from `process.env`
   - If either is missing, throw `new ApiError(ERROR_CODES.EXTERNAL_API_ERROR, 'Netatmo proxy not configured: missing NETATMO_PROXY_URL or NETATMO_PROXY_API_KEY', HTTP_STATUS.INTERNAL_SERVER_ERROR)`
   - Create AbortController with `timeout` (default 15000ms)
   - `fetch(baseUrl + endpoint, { headers: { 'X-API-Key': apiKey }, signal })`
   - On AbortError: throw `ApiError.timeout('Netatmo proxy timeout')`
   - On non-ok response: try to parse body as RFC 9457 (`RFC9457ProblemDetail`). If body has `detail` field, use it as error message. Map status: 401 -> UNAUTHORIZED, 503 -> SERVICE_UNAVAILABLE, else EXTERNAL_API_ERROR with HTTP_STATUS.BAD_GATEWAY
   - On success: return `await response.json() as T`
   - Always `clearTimeout` in finally

2. Convenience wrappers (thin, call netatmoProxyGet internally):
   - `getProxyHomestatus(): Promise<NetatmoProxyHomestatusResponse>` — calls `netatmoProxyGet('/homestatus')`
   - `getProxyHomesdata(): Promise<NetatmoProxyHomesdataResponse>` — calls `netatmoProxyGet('/homesdata')`

Pattern reference: Similar to fritzboxClient.ts `request()` method but simpler — no JWT token management, no retry on 401, just API key header.
