Root cause: `fetchData` is memoized with `useCallback([enrichDevicesWithCategories])`. Inside it, the FRITZBOX_TIMEOUT error handler and the network-error catch block read `bandwidth` and `wan` directly from the closure:

```typescript
} else if (errorData.code === 'FRITZBOX_TIMEOUT') {
  setStale(true);
  if (!bandwidth && !wan) {  // stale — always null, callback was never recreated
    setError({ type: 'generic', ... });
  }
}
// ...
} catch (err) {
  setStale(true);
  if (!bandwidth && !wan) {  // stale — same problem
    setError({ type: 'generic', ... });
  }
}
```

Because `bandwidth` and `wan` are not in the useCallback deps array, the callback captures them at creation time (both `null`). After the first successful fetch, `setBandwidth(bw)` updates state but does NOT recreate `fetchData`. The guard always evaluates `null && null = true` and wrongly sets a generic error even when cached data exists.

Fix: Add two refs that track current values, then use them in both guard conditions.

Change 1 — Add two refs next to the existing refs block (after `enrichedMacsRef`, before the useEffect):
```typescript
// Add these two lines after: const enrichedMacsRef = useRef<Set<string>>(new Set());

const wanRef = useRef<WanData | null>(null);
```

Change 2 — After `setBandwidth(bw)` in the success path (inside `if (bw)` block around line 221), add:
```typescript
bandwidthRef.current = bw;
```

Change 3 — After `setWan(wanData.wan || null)` (around line 232), add:
```typescript
wanRef.current = wanData.wan || null;
```

Change 4 — In the FRITZBOX_TIMEOUT handler (around line 187), replace `!bandwidth && !wan` with refs:
```typescript
// BEFORE:

// AFTER:
```

Change 5 — In the network-error catch block (around line 278), replace `!bandwidth && !wan` with refs:

Do NOT add `bandwidth` or `wan` to the useCallback dependency array (would cause unnecessary re-renders). Do NOT change any other logic. The pattern of using refs for stable cross-render reads is already established in this file (healthRef, consecutiveReadingsRef, enrichedMacsRef).
