# Quick Task 260423-n6i: Jest Performance & Resource Fix

## Problem

During the phase 170 chain execution I observed:

1. `jest-haste-map: duplicate manual mock found: next-server / Text` — Jest was indexing `__mocks__/` both in repo root AND inside `.claude/worktrees/agent-*/` directories.
2. `jest-haste-map: Haste module naming collision: pannello-stufa` — Jest was indexing two `package.json` files (root + worktree copy).
3. `npm test` (full suite) ran for 25+ minutes at 90% CPU on one core, with 4 worker processes total eating RAM.
4. Every GSD wave verification triggers `npm test` which re-runs the entire 300+ file suite.

Root cause in `jest.config.ts`:
- `testPathIgnorePatterns` excludes `/tests/` (Playwright) and `__utils__/` but NOT `.claude/worktrees/` or `.planning/`
- No `modulePathIgnorePatterns` → haste-map still indexes excluded dirs for module resolution (exclude pattern is different category)
- No `watchPathIgnorePatterns`
- No `maxWorkers` cap → Jest claims every core by default
- Only `test`, `test:watch`, `test:coverage`, `test:ci` scripts — no ergonomic scoped targets for iterative dev

## Tasks

### Task 1: Harden `jest.config.ts` against worktrees and cap workers

<read_first>
- jest.config.ts
</read_first>

<action>
Add to `customJestConfig`:
- Extend `testPathIgnorePatterns` with `<rootDir>/.claude/`, `<rootDir>/.planning/`, `<rootDir>/.claude/worktrees/`.
- Add `modulePathIgnorePatterns: ['<rootDir>/.claude/', '<rootDir>/.planning/', '<rootDir>/.next/']` so haste-map stops indexing these dirs (this is the fix for the duplicate-mock and haste-collision warnings).
- Add `watchPathIgnorePatterns: ['<rootDir>/.claude/', '<rootDir>/.planning/', '<rootDir>/node_modules/', '<rootDir>/.next/']`.
- Add `maxWorkers: process.env.CI ? 2 : '50%'` — respects existing `test:ci` override semantics, halves local CPU usage.
</action>

<acceptance_criteria>
- `grep -c "'.claude/worktrees" jest.config.ts` ≥ 1 (via the `.claude/` pattern covering it) — verified by `grep "'<rootDir>/.claude/'" jest.config.ts`
- `grep "modulePathIgnorePatterns" jest.config.ts` → match found
- `grep "maxWorkers" jest.config.ts` → match found
- After edits: run `npx jest __tests__/hooks/useLogin.test.ts` — no `duplicate manual mock` warning printed, exit 0
</acceptance_criteria>

### Task 2: Add scoped npm test scripts

<read_first>
- package.json
</read_first>

<action>
Add to `scripts` section:
- `"test:changed": "jest --onlyChanged --passWithNoTests"` — run tests for files touched in current working tree/staged commits
- `"test:unit": "jest __tests__/lib __tests__/hooks __tests__/utils"` — pure logic/hook tests
- `"test:api": "jest __tests__/api"` — API route tests only
- `"test:components": "jest __tests__/app/components"` — UI component tests
- `"test:pages": "jest __tests__/app --testPathIgnorePatterns='__tests__/app/components'"` — page/route tests
- `"test:quick": "jest --bail --passWithNoTests"` — bail on first failure for fast iteration

Leave `test`, `test:random`, `test:watch`, `test:coverage`, `test:ci` untouched.
</action>

<acceptance_criteria>
- `grep "test:changed" package.json` → match
- `grep "test:unit" package.json` → match
- `grep "test:api" package.json` → match
- `grep "test:components" package.json` → match
- `grep "test:pages" package.json` → match
- `grep "test:quick" package.json` → match
- `npm run test:changed --silent -- --listTests 2>/dev/null` exits 0 (even when no changes, `--passWithNoTests` keeps it non-error)
</acceptance_criteria>
