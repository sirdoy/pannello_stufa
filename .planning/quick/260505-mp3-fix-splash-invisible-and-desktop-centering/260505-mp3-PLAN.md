# Quick 260505-mp3 — Plan

## Problem 1: Reload renders blank but clicks open Sheet

`SplashGate` ([app/components/EmberGlass/SplashGate.tsx:43](../../../app/components/EmberGlass/SplashGate.tsx:43)) keeps wrapper at `opacity: 0` until `ready === true`. `ready` only flips inside `<Splash onDone>` — but `<Splash>` only mounts when `shouldShowSplash` is true, which excludes `shownThisSession === true`.

On reload (sessionStorage flag set) → splash skipped → `ready` stuck false → wrapper invisible forever. `BottomTabBar` is sibling of `ClientProviders` (Phase kn6), so it stays visible/clickable → tap opens Sheet.

Same trap for logged-out / public routes: `!user` makes `shouldShowSplash` false but `ready` never gets set.

**Fix**: in the hydration `useEffect`, if `shownThisSession` is true, eagerly `setReady(true)`. Add a second effect that sets `ready=true` once auth resolves to no user (`!isLoading && !user`).

## Problem 2: Desktop bar not centered

`globals.css:368-376` has the desktop centering rule (`left: 50%; transform: translateX(-50%); width: 480px`). `BottomTabBar.tsx:59-60` sets inline `left: 12, right: 12`. Inline styles beat stylesheet rules → desktop media query never applies.

**Fix**: drop `left/right` from the inline style. Add `left: 12px; right: 12px;` to the base `[data-bottom-tab="true"]` rule in globals.css so mobile keeps full-width pill, desktop overrides via existing media query.
