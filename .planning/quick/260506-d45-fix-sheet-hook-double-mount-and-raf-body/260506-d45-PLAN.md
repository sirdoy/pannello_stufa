Surgical edit to the scroll-lock `useEffect` at `Sheet.tsx:49-65`. Goal: remove the synchronous reflow at open time by writing `document.body.style` mutations (and their cleanup restore) inside `requestAnimationFrame`, so they happen after the slide-in transform's first paint. Preserve the `lockedScrollY` ref capture semantics exactly (still synchronous on the open edge — only the style writes defer).

**Replacement structure (per RESEARCH §Fix-D):**

```tsx
useEffect(() => {
  if (!open) return;
  // Capture scrollY synchronously — must run before browser repaints.
  // Otherwise the body becoming `position:fixed` on a later frame would
  // already see scrollY=0 (the document jumped to top during the rAF gap).
  lockedScrollY.current = window.scrollY;
  incrementSheetCount();        // synchronous — global sheet-open counter
                                // (used by AmbientBg pause CSS rule
                                //  shipped in 260505-otx; cannot be deferred
                                //  or AmbientBg keeps painting one extra frame).

  const setRaf = requestAnimationFrame(() => {
    document.body.style.position = 'fixed';
    document.body.style.top = `-${lockedScrollY.current}px`;
    document.body.style.width = '100%';
    document.body.style.overflow = 'hidden';
  });

  return () => {
    cancelAnimationFrame(setRaf);
    // Restore writes also rAF-deferred so the close transition starts
    // its slide-out before layout invalidation. window.scrollTo MUST
    // run AFTER the position:'' write or the scroll restore happens
    // on a fixed-positioned body and is no-op'd.
    const restoreRaf = requestAnimationFrame(() => {
      document.body.style.position = '';
      document.body.style.top = '';
      document.body.style.width = '';
      document.body.style.overflow = '';
      window.scrollTo(0, lockedScrollY.current);
    });
    decrementSheetCount();      // synchronous — symmetric with increment.
    // We don't track the cancel id for restoreRaf because the cleanup
    // function is only invoked once per [open=true → open=false] cycle;
    // a subsequent re-open re-runs the effect and re-captures scrollY,
    // and the restoreRaf write to '' is idempotent if a new open arrives
    // mid-flight (next setRaf will overwrite the cleared values).
  };
}, [open]);
```
