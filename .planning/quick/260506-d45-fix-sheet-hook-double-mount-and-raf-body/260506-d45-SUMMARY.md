# quick-260506-d45 Plan 01: Sheet hook double-mount + rAF body-style writes тАФ Summary

One-liner: rAF-defer Sheet scroll-lock writes (Fix D) and lift device hooks from sheet bodies to parent cards via a presentational + SelfFetch-wrapper pattern (Fix B).

## Edits Made

### Task 1 тАФ Defer Sheet scroll-lock body-style writes via rAF (Fix D) тАФ commit 0fccc9c5

`app/components/EmberGlass/Sheet.tsx`

The scroll-lock `useEffect` was rewritten so the synchronous reflow of `position: fixed` + the cleanup restore happen inside `requestAnimationFrame`. Frame budget for the slide-in's first paint stays clear; layout cost lands on frame +1 instead of fighting the 400ms transform.

Synchronous on the open edge (must NOT defer):

- `lockedScrollY.current = window.scrollY` тАФ captured before any subsequent paint, otherwise the document jumps to top during the rAF gap and we'd lock a useless 0
- `incrementSheetCount()` / `decrementSheetCount()` тАФ drives the AmbientBg pause CSS rule shipped in 260505-otx; deferring would let AmbientBg keep painting for one extra frame

Deferred via rAF:

- `document.body.style.{position,top,width,overflow}` (open writes + cleanup restore)
- `window.scrollTo(0, lockedScrollY.current)` тАФ must run AFTER the position:'' write, otherwise the scroll restore happens on a fixed-positioned body and is a no-op

Cleanup uses `cancelAnimationFrame` on the open-edge rAF id; the restore rAF is fire-and-forget (idempotent w.r.t. mid-flight re-open).

`app/components/EmberGlass/__tests__/Sheet.test.tsx`

Added a `flushRaf()` helper that drains one rAF callback inside `act()`. Two existing scroll-lock tests now `await flushRaf()` before asserting body styles. New test: regression guard verifying `requestAnimationFrame` fires on both open and close (jest spies `window.requestAnimationFrame` and asserts the call count grows on each transition).

### Task 2 тАФ Lift device hooks from 5 sheet bodies to parent cards (Fix B) тАФ commit a37ad4e1

Each of the 5 device sheets (Stove, Climate, Lights, Sonos, Plugs) now exports two named components:

- `*Sheet({...props})` тАФ presentational, REQUIRED `data` + `cmds` props; ZERO `use*Data` / `use*Commands` invocations in the body
- `*SheetSelfFetch()` тАФ zero-prop wrapper that calls the same hooks the body used to call internally and renders the prop-based body

Production cards now own the single hook mount and thread results into the sheet via props:

- `StoveCard` adds `useStoveCommands` + `useRouter`; renders `<StoveSheet stoveData={stove} cmds={cmds} onNavigate={(p) => router.push(p)} />`
- `ClimateCard` adds `useThermostatCommands`; renders `<ClimateSheet data={data} cmds={cmds} />`
- `LightsCard` was already mounting both hooks; renders `<LightsSheet lightsData={lightsData} cmds={cmds} />` (pure waste-removal тАФ the sheet's second copy was the duplicate)
- `SonosCard` adds `useSonosCommands` + a local `setCommandError` sink; renders `<SonosSheet sonosData={sonosData} cmds={cmds} />`
- `TuyaCard` adds `useTuyaCommands`; renders `<PlugsSheet tuyaData={tuyaData} cmds={cmds} />`

Hook mount accounting per device dropped 2 тЖТ 1 while a sheet is open. Worst-offender Sonos (~20 fan-out fetches duplicated on every open) is fully deduplicated.

Card jest specs updated to mock the newly-card-side `use*Commands` hooks (parity with the existing sheet-body mock pattern). The mock factories return jest.fn() stubs for each handler; this avoids pulling in `ToastProvider` / `useRetryableCommand` / etc. that the real commands hooks depend on.

`app/components/EmberGlass/sheets/index.ts` тАФ barrel exports both `*Sheet` and `*SheetSelfFetch` for each of the 5 sheets, plus their `*SheetProps` types.

### Task 3 тАФ Rewire sheet jest tests + Section10SheetGallery to consume new sheet API тАФ commit b567b4fd

Locks in the prop-based contract; if a future regression re-introduces `use*Data` into the body, the body-grep regression in 260506-d45 PLAN done-criteria catches it.
