### Drift Commit 2 — 260504-kn6: BottomTabBar hoisted outside ClientProviders

**File changed:** `app/layout.tsx` — `<BottomTabBar />` moved from inside `<ClientProviders>` to a sibling element after `</ClientProviders>`, as a direct child of `<body>`.

**Cross-phase checks:**

1. **NAV-01..04 functional routing**: WIRED — `usePathname()` (next/navigation, provided by Next.js App Router at framework level, not by ClientProviders) still works. All 4 routes (`/`, `/stanze`, `/automazioni`, `/altro`) resolve to existing `page.tsx` files.

2. **NAV-02 accent live-paint**: WIRED — BottomTabBar consumes `var(--accent)` via inline styles. `Section01Hue.tsx:29` calls `document.documentElement.style.setProperty('--accent', value)` on `:root` — a DOM global that is independent of React tree position. No regression.

3. **NAV-03 body[data-sheet-open] hide rule**: WIRED — `globals.css:378` selector `body[data-sheet-open="true"] [data-bottom-tab="true"]` is a CSS descendant selector from `<body>`. BottomTabBar is now a direct child of `<body>`, which is still a descendant of `<body>`. Selector continues to match. `SheetCounter.ts` still toggles `document.body.dataset.sheetOpen` and `Sheet.tsx:56,63` still calls `incrementSheetCount()` / `decrementSheetCount()` in its `[open]` useEffect. Chain: Sheet open → incrementSheetCount → body.dataset.sheetOpen="true" → CSS translateY(140%) hides BottomTabBar.

4. **NAV-04 safe-area inset**: WIRED — `BottomTabBar.tsx:58` still uses `bottom: 'calc(8px + env(safe-area-inset-bottom))'`.

5. **Context dependency check**: WIRED — BottomTabBar imports only `usePathname` (next/navigation), `Link` (next/link), lucide-react icons, and `Pressable` (pure component with no context dependencies). No Auth0, WebSocket, Version, Toast, or any other ClientProviders-scoped context is consumed. The hoist is safe.

6. **Root cause fixed**: WIRED — `SplashGate.tsx:65` wraps `{children}` in a div with `transform: scale(0.97→1)`. Per CSS spec, any non-`none` transform on an ancestor creates a containing block for `position: fixed` descendants. With BottomTabBar now outside that wrapper, it correctly anchors to the viewport.

7. **NavbarConnectionStatusChip not affected**: WIRED — the WS chip remains inside `ClientProviders` (layout.tsx:72) where its `useWebSocketContext()` dependency is satisfied. Its `data-ws-chip="true"` / `globals.css:397-400` hide-on-sheet rule is unaffected.

**Verdict:** WIRED. 260504-kn6 correctly fixes a position:fixed containing-block bug. No NAV-* requirement regressed; Sheet→BottomTabBar hide chain intact.

### Warnings (non-blocking)

**WARNING-01** — BottomTabBar context isolation (NAV-01): BottomTabBar is now rendered outside `ClientProviders` and therefore outside `Auth0Provider`, `WebSocketContext`, `VersionProvider`, etc. This is currently safe because BottomTabBar has zero context dependencies. However, if a future phase adds context-dependent logic to BottomTabBar (e.g., showing a WS connection badge inline), developers must remember to either provide the required context at layout level or move BottomTabBar back inside an appropriate provider boundary. This is a structural constraint to document, not a current defect.
